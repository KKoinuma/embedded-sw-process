## 3. テスト種別と実施方針

### 3.1 単体テスト

- **目的:** 各モジュールの機能が要件を満たすことを確認
- **実施者:** 開発担当
- **ツール:** [Unity / GoogleTest / 等]
- **合否基準:**
  - 全テストケース Pass
  - C1 カバレッジ ≥ 90%
  - 静的解析 Error 0件

### 3.2 結合テスト

- **目的:** モジュール間インタフェースの正常動作を確認
- **実施者:** 開発担当 (SW リーダ監督)
- **合否基準:** 全テストケース Pass

### 3.3 システムテスト

- **目的:** システム全体として要件を満たすことを確認
- **実施者:** QA 担当
- **合否基準:**
  - 全必須テストケース Pass
  - 未解決バグ (重大度 High 以上) 0件
