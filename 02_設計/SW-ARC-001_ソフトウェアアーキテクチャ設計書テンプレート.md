## 2. アーキテクチャ概要

### 2.1 設計方針

1. **安全性優先**: SIL 対応機能は独立したモジュールとして分離する
2. **疎結合**: モジュール間は定義されたインタフェースのみで通信する
3. **静的設計**: 動的メモリ割り当て禁止。全バッファをコンパイル時に確定する
4. **防御的プログラミング**: 全入力値を範囲チェックする

### 2.2 レイヤ構成

```
┌─────────────────────────────────────────┐
│          アプリケーション層              │
│  [制御ロジック] [状態管理] [診断機能]    │
├─────────────────────────────────────────┤
│          サービス層                      │
│  [通信サービス] [ログサービス] [タイマ]  │
├─────────────────────────────────────────┤
│          HAL (ハードウェア抽象化層)       │
│  [GPIO] [UART] [SPI] [ADC] [WDT]        │
├─────────────────────────────────────────┤
│          RTOS / Bare Metal               │
└─────────────────────────────────────────┘
```

## 3. モジュール設計

### 3.1 モジュール一覧

| モジュール名 | 責務 | SIL | 依存モジュール |
|---|---|---|---|
| main | システム初期化・メインループ | - | 全モジュール |
| ctrl | 制御ロジック | SIL2 | hal_gpio, svc_timer |
| comm | 通信処理 | SIL1 | hal_uart |
| diag | 自己診断 | SIL2 | hal_wdt, hal_gpio |
| hal_gpio | GPIO 制御 | - | - |
| hal_uart | UART 制御 | - | - |
| svc_timer | タイマサービス | - | hal_timer |
| svc_log | ログサービス | - | hal_uart |

### 3.2 モジュール間インタフェース

| 提供モジュール | 利用モジュール | インタフェース | 形式 |
|---|---|---|---|
| hal_gpio | ctrl | GPIO_Write() / GPIO_Read() | 関数呼び出し |
| svc_timer | ctrl | Timer_GetTick() | 関数呼び出し |
| comm | アプリ層 | CommRxQueue / CommTxQueue | リングバッファ |

## 4. タスク・スケジューリング設計

### 4.1 タスク一覧 (RTOS 使用時)

| タスク名 | 優先度 | 周期/イベント | スタックサイズ | 担当処理 |
|---|---|---|---|---|
| Task_Safety | 最高 | 1 ms | 512 B | 安全監視・WDT リフレッシュ |
| Task_Control | 高 | 10 ms | 1024 B | 制御ロジック |
| Task_Comm | 中 | イベント | 512 B | 通信処理 |
| Task_Log | 低 | 100 ms | 256 B | ログ出力 |

### 4.2 割り込み設計

| 割り込み | 優先度 | 処理内容 | 最大処理時間 |
|---|---|---|---|
| UART_RX | 高 | 受信データをバッファへ格納 | ≤ 10 μs |
| Timer_1ms | 最高 | タイムベース更新 | ≤ 5 μs |
| ADC_EOC | 中 | ADC 結果読み出し | ≤ 20 μs |

## 5. データ設計

### 5.1 共有データ管理方針

- タスク間共有変数は排他制御 (Mutex / 割り込み禁止) を必ず実施する
- グローバル変数はモジュール内スコープ (`static`) を原則とする
- センサ値・制御値は構造体で管理し、型安全性を確保する

### 5.2 主要データ構造

```c
/* システム状態 */
typedef enum {
    SYS_STATE_INIT    = 0,
    SYS_STATE_NORMAL  = 1,
    SYS_STATE_WARNING = 2,
    SYS_STATE_FAULT   = 3
} SysState_t;

/* センサデータ */
typedef struct {
    uint32_t timestamp_ms;
    int32_t  value_raw;
    float    value_calibrated;
    bool     valid;
} SensorData_t;
```

| エラー種別 | 検出方法 | 対応動作 |

| 通信エラー | CRC チェック・タイムアウト | 再送・上位通知 |

| メモリ異常 | スタック境界マーカ | フォルトハンドラ → リセット |
